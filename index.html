Required Coda tables

Create a new Coda doc and add these four tables exactly as named:

Table 1 — Reservations
Columns: Owner (text), Type (text), StartDate (date), EndDate (date), Status (text), Quarter (text), Note (text), Guests (text)
One row per stay — a contiguous trip is stored as a single StartDate–EndDate range. Guests is optional (names invited to join or stand in for the owner); the invite UI appears automatically once the column exists.

Table 2 — QuarterState
Columns: Quarter (text), Phase (text)

Table 3 — Config
Columns: Key (text), Value (text)
Add one row: Key = StartDate, Value = 2026-08-01

Table 4 — Users
Columns: Name (text), Password (text), Role (text), DaysPerQuarter (number)
One row per person who can sign in. Role is one of owner, contributor, other_family, or guest. A guest can sign in and view the calendar but cannot book any days — give them a Password just like everyone else. other_family is the former “guest” role and can still book floating days when a quarter is open to all.
